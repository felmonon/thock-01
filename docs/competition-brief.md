## Deployment constraints

- Webflow Cloud supports Astro 5 or newer; this project uses Astro 6.
- Node 22+ and npm are required.
- Webflow Cloud currently supports npm, which matches this repository’s lockfile.
- A standalone app on a new domain avoids mount-path asset-prefix complexity.
- `webflow.json` declares the `astro` framework.
- The deployed URL must be opened and tested independently after the deployment succeeds.

## Project non-negotiables

- One-sentence concept: every visitor creates and replays a unique mechanical-keyboard soundprint.
- Touch and physical-keyboard input must both create a soundprint.
- Sound must never autoplay and must have a visible mute control.
- Reduced motion must preserve the product story and signal visualization.
- No copyrighted product imagery, audio samples, or logos are used.
- The production page must have no uncaught errors, missing assets, or horizontal overflow.
- The final Google Form submission is intentionally outside automated deployment scope until the user explicitly approves sending it.

## Unknowns

The official page does not publish a scoring formula, total submission count, detailed eligibility language, or a public entry gallery. The submission form’s responses are private. Any win probability is therefore an estimate, not a promise.
